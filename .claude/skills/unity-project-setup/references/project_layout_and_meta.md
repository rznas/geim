# Project layout, .meta files, Library/, and .gitignore

Grounded in `references/api/default-directories.md`, `SpecialFolders.md`,
`AssetMetadata.md`, and `Versioncontrolintegration.md`. Doc-sourced; not
compile-tested here.

## What Unity creates on first open

From `default-directories.md`, a Unity project's root contains:

| Folder / file | Commit? | What it is |
|---------------|:------:|------------|
| `Assets/` | **yes** | All your scripts, scenes, textures, models, audio. The Project window shows this. |
| `Packages/manifest.json` | **yes** | The project manifest — which UPM packages to load (see `packages_and_asmdef.md`). |
| `ProjectSettings/` | **yes** | Project-specific settings (graphics, input, tags, build, physics). |
| `Library/` | **NO** | "A local cache of imported assets and metadata … unique to your computer." Regenerated; can be deleted to fix a corrupt project. |
| `Temp/` | **NO** | "Temporary data, which gets cleared every time you close Unity." |
| `Logs/` | **NO** | Editor/import log files. |
| `UserSettings/` | **NO** | "Settings for your local version of the project … your preferred Editor layout." Excluding it avoids overwriting teammates' preferences. |

The manual explicitly says to *exclude* `Library/`, `Temp/`, and `UserSettings/`
from version control. Everything else above is committed.

`new_unity_project.sh` writes `Assets/`, `Packages/manifest.json`,
`ProjectSettings/` (ProjectVersion.txt + a minimal ProjectSettings.asset), and a
Unity `.gitignore` — the portable skeleton. Unity regenerates `Library/`, the
rest of `ProjectSettings/`, the `.csproj` and the `.sln` on first open. There is
no editor-less way to produce those (`cli_and_batchmode.md`).

## `.meta` files — commit every single one

This is the gotcha that silently corrupts projects. From `AssetMetadata.md`:

> Unity creates `.meta` files for each folder and file in your project's
> `Assets` folder and stores the `.meta` file in the same location as the asset
> file.

Each `.meta` holds the asset's **unique GUID** plus all its import settings.
Unity references assets by that GUID internally so it can move/rename without
breaking links. Therefore:

- **A `.meta` must travel with its asset.** If you move/rename an asset *outside*
  Unity, move/rename its `.meta` too. (Inside the Editor, Unity does it for you.)
- **Losing a `.meta` breaks every reference to that asset** (verbatim concern from
  the doc): a texture's lost `.meta` orphans every material that used it; a
  script's lost `.meta` turns every GameObject/Prefab using it into an
  "unassigned script component." You then must reassign by hand.
- **Folders have `.meta` files too**, and they matter for empty folders under
  VCS: if Unity sees a folder `.meta` with no folder it recreates the folder;
  if it sees an empty folder with no `.meta` it deletes the folder. So commit
  folder `.meta`s.

`.meta` files are hidden in the Project window and often hidden by the OS file
browser, which is exactly why people forget to add them to git. **`.gitignore`
must not ignore `*.meta`.**

### Visible vs hidden meta files

`Versioncontrolintegration.md`: the Editor's **Version Control** mode defaults to
**Visible meta files** — "the default setting … manage the source Assets and
metadata … with a version control system of your choice." Use this with git.
**Hidden meta files** only hides them in your OS file explorer; either way Unity
does not show `.meta` in the Project view. For git, leave it on Visible.

## Reserved folder names (SpecialFolders.md)

Certain `Assets/` subfolder names have special compilation/runtime meaning — do
not use them casually:

| Name | Effect |
|------|--------|
| `Editor` | Scripts here compile into `Assembly-CSharp-Editor` and are **stripped from Player builds**. Put `-executeMethod` targets and custom tools here. MonoBehaviours in `Editor` can't be attached to GameObjects. |
| `Resources` | Assets here load at runtime via `Resources.Load`. They **always ship in the build** even if unreferenced, inflating build size — use sparingly. |
| `StreamingAssets` | Files kept in original format, available at runtime for streaming. Root of `Assets/` only. |
| `Plugins` | Third-party / native plugins. |
| `Gizmos`, `Editor Default Resources` | Editor-only icon/resource folders. |

Also: Unity ignores during import any file/folder that is hidden, starts with
`.`, ends with `~`, is named `cvs`, or ends in `.tmp`. The scaffold's source
layout (`Scripts/`, `Scenes/`, `Art/`, `Audio/`, plus a reserved `Editor/`)
avoids all reserved-name surprises.

## The .gitignore template

`assets/unity.gitignore` is the copy-paste version (the scaffold inlines an
equivalent). It ignores `Library/`, `Temp/`, `Logs/`, `UserSettings/`, `obj/`,
build output, and the generated `*.csproj` / `*.sln` (regenerated by the IDE
package on open), while leaving `Assets/`, `Packages/`, `ProjectSettings/`, and
all `*.meta` files tracked. Note: the Unity *manual* never prints a `.gitignore`
verbatim — this template is synthesised from the "exclude from version control"
statements in `default-directories.md` and the "commit your .meta" rule in
`AssetMetadata.md`.
