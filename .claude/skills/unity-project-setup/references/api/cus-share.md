# Share your package

When you have finished developing your package, you can share it with others to use.

Choose from the following methods to share your package:

| **Type** | **Description** |
| --- | --- |
| **Compressed file** | You can share a zip file to other Unity users. That way, they can decompress the zip file to a local folder on their own computer and install the package from disk. |
| **Tarball** | You can share a tarball to other Unity users. That way, they can install the package from the local tarball directly. For information about creating a tarball from a **UPM package** you created, refer to Export and sign your UPM package. |
| **Git URL** | You can share a link to your Git repository, using one of the supported protocols. Then users can install your package using its Git URL, because the Unity Package Manager can fetch packages from services that host Git repositories, such as GitHub and GitLab. |
| **Scoped Registry** | You can set up a package registry server to host your package and then publish it to that registry. For more information, refer to Host a scoped registry.  Your package’s consumers can then configure their project settings to Use a scoped registry to fetch your custom packages from your package registry. |
