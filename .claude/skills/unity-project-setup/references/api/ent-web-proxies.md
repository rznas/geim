# Set up Unity for web proxies

If your organization gets internet access through a web proxy, you can configure the web proxy to allow access to Unity applications. This section contains details on how to set up Unity and your web proxy for a reliable service.

| **Topic** | **Description** |
| --- | --- |
| **Use Unity through web proxies** | Prepare your environment so Unity applications can communicate through a web proxy. |
| **Enable automatic proxy configuration** | Configure client computers to use Unity’s automatic proxy configuration feature. |
| **Store credentials for automatic proxy configuration (Windows)** | Store authentication credentials on Windows for web proxies that require basic authentication. |
| **Store credentials for automatic proxy configuration (macOS)** | Store authentication credentials on macOS for web proxies that require basic authentication. |
| **Trusting the web proxy security certificate** | Configure Unity to trust the SSL/TLS certificate of a web proxy that uses SSL inspection. |
| **Use environment variables to identify your web proxy** | Set environment variables to identify your web proxy manually. |
| **Create a command file to set environment variables and open applications** | Create an executable file to launch the Unity Hub with specific environment variables in the same session. |
| **Define exceptions on your web proxy** | Allow specific Unity URLs through the web proxy without authentication or SSL inspection. |
